# 后端需求补充

## 后端目标
网站需要有完整询盘后端，用来接收海外批发客户提交的信息，并保存到数据库，同时发送邮件/通知给管理员。

## 后端功能

### 1. 询盘提交 API

接口：

POST /api/inquiry

前端表单提交字段：

{
  name: string,
  email: string,
  whatsapp?: string,
  country?: string,
  productInterest?: string,
  quantity?: string,
  message?: string
}

要求：
- 校验必填字段：name、email、message
- 校验 email 格式
- 防止空提交
- 防止重复频繁提交，可加简单 rate limit
- 提交成功后返回：

{
  success: true,
  message: "Inquiry submitted successfully"
}

失败返回：

{
  success: false,
  message: "Error message"
}

## 2. 数据库

建议使用 PostgreSQL，部署可用 Supabase / Neon / Vercel Postgres。

数据表：inquiries

字段：

id  
name  
email  
whatsapp  
country  
product_interest  
quantity  
message  
status  
created_at  
updated_at  

status 默认值：new

## 3. 管理后台

需要一个简单后台页面：

/admin/inquiries

功能：
- 查看所有询盘
- 按时间倒序排列
- 查看客户姓名、邮箱、WhatsApp、国家、产品、数量、留言
- 可以修改状态：
  - new
  - contacted
  - quoted
  - closed
  - spam

后台需要简单登录保护。

可以先用环境变量设置管理员密码：

ADMIN_PASSWORD=xxxxxx

/admin 登录后才能查看询盘。

## 4. 邮件通知

客户提交询盘后，自动发送邮件给管理员。

管理员邮箱环境变量：

ADMIN_EMAIL=your@email.com

邮件标题：

New Wholesale Inquiry from {{name}}

邮件内容包含：
- Name
- Email
- WhatsApp
- Country
- Product Interest
- Quantity
- Message
- Submit Time

邮件服务可用：
- Resend
- SendGrid
- Nodemailer SMTP

优先使用 Resend，配置简单。

## 5. WhatsApp 引导

提交成功后，前端提示客户：

Thank you. We have received your inquiry. You can also contact us directly on WhatsApp for a faster quote.

并显示 WhatsApp 按钮。

## 6. 环境变量

需要支持：

DATABASE_URL=
ADMIN_EMAIL=
ADMIN_PASSWORD=
RESEND_API_KEY=
NEXT_PUBLIC_WHATSAPP_NUMBER=

## 7. 安全要求

- API 不允许直接暴露数据库错误
- 表单字段要做基础清洗
- 后台页面需要登录保护
- 不要把管理员密码写死在代码里
- 不要把 API Key 写死在前端
- 所有密钥通过 .env.local 配置

## 8. 后端技术建议

如果使用 Next.js：
- 使用 Next.js Route Handler 写 API
- 使用 Prisma ORM 操作数据库
- 使用 PostgreSQL
- 使用 Resend 发邮件
- 使用 cookie/session 做简单后台登录

## 9. Prisma Schema 示例

model Inquiry {
  id              String   @id @default(cuid())
  name            String
  email           String
  whatsapp        String?
  country         String?
  productInterest String?
  quantity        String?
  message         String
  status          String   @default("new")
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}
