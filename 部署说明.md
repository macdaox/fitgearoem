# Cloudflare 后台部署说明

这份文档按你的实际方式写：在 Cloudflare 后台绑定 GitHub 仓库，然后 Cloudflare 自动构建和部署。

项目仓库：

```text
https://github.com/macdaox/fitgearoem
```

## 重要说明

这个网站不是纯静态页面，它有：

- 后台登录
- 后台内容管理
- 图片上传
- 询盘提交接口
- 询盘后台管理
- DeepSeek 翻译接口

所以部署时不要按普通静态 Pages 项目处理。Cloudflare 后台的产品入口可能叫 `Workers & Pages`，但这个项目实际要用 OpenNext 跑完整 Next.js 后台能力。

如果 Cloudflare 后台让你选择项目类型，优先选择支持 `Workers / OpenNext / Full-stack Next.js` 的方式。不要选择只输出静态文件的配置，否则后台和 API 会失效。

## 1. GitHub 仓库

代码已经推送到：

```text
https://github.com/macdaox/fitgearoem
```

以后你本地改完后：

```bash
git add .
git commit -m "修改说明"
git push
```

Cloudflare 后台绑定 GitHub 后，会自动拉取最新代码部署。

## 2. Cloudflare 后台绑定 GitHub

进入 Cloudflare 控制台：

```text
Cloudflare Dashboard -> Workers & Pages
```

然后：

1. 点击创建应用
2. 选择连接 GitHub
3. 授权 Cloudflare 访问 GitHub
4. 选择仓库：`macdaox/fitgearoem`
5. 分支选择：`main`

## 3. 构建配置

在 Cloudflare 后台配置构建时，填写：

```text
Framework preset: Next.js
```

如果后台有 Build command / Deploy command 选项，填写：

```text
npm run deploy
```

如果它分开要求 Build command，可以填写：

```text
npm run cf:build
```

如果它要求 Output directory，填写：

```text
.open-next/assets
```

如果 Cloudflare 后台明确提示这是 `Workers Builds / OpenNext`，通常不需要你手动填写 Output directory，以后台提示为准。

## 4. Cloudflare KV

KV 用来保存：

- 后台编辑的网站文字
- 后台编辑的联系方式
- 询盘记录
- 询盘状态

KV 现在是可选项。你可以用 KV 保存后台内容和询盘，也可以直接用下面第 5 步的 R2 API 保存。

如果你想用 KV，在 Cloudflare 后台创建 KV：

```text
Workers & Pages -> KV -> Create namespace
```

命名：

```text
SITE_DATA
```

创建后，把这个 KV namespace 绑定到当前项目。

绑定名称必须是：

```text
SITE_DATA
```

当前 `wrangler.jsonc` 里没有写死 KV ID，避免 Cloudflare 自动部署时因为占位 ID 报错。你可以在 Cloudflare 后台 UI 里绑定 `SITE_DATA`。

## 5. Cloudflare R2

R2 用来保存后台上传的图片，也可以保存后台内容和询盘 JSON。

在 Cloudflare 后台创建一个 R2 bucket：

```text
jump-rope-site-images
```

这个 bucket 用于：

- 后台上传的网站图片
- 后台编辑的网站内容
- 客户询盘记录

如果你愿意用 Cloudflare 绑定，也可以在项目里绑定：

```text
Binding name: SITE_IMAGES
Bucket: jump-rope-site-images
```

但现在项目也支持通过 R2 API 环境变量上传和保存数据，所以这个绑定不是必须的。

## 6. R2 图片公开访问

后台上传图片后，前台页面需要能访问这些图片，所以 `jump-rope-site-images` 要配置公开访问。

在 Cloudflare 后台：

```text
R2 Object Storage -> jump-rope-site-images -> Settings
```

然后开启公开访问，或者绑定自定义域名。

拿到公开地址后，后面要填到环境变量：

```text
CLOUDFLARE_R2_PUBLIC_URL
```

例子：

```text
https://images.yourdomain.com
```

结尾不要加 `/`。

## 7. 环境变量

在 Cloudflare 项目后台找到：

```text
Settings -> Environment Variables
```

添加这些变量：

```text
ADMIN_EMAIL
ADMIN_PASSWORD
RESEND_API_KEY
DEEPSEEK_API_KEY
DEEPSEEK_MODEL
NEXT_PUBLIC_WHATSAPP_NUMBER
NEXT_PUBLIC_SITE_URL
CLOUDFLARE_R2_ACCOUNT_ID
CLOUDFLARE_R2_ACCESS_KEY_ID
CLOUDFLARE_R2_SECRET_ACCESS_KEY
CLOUDFLARE_R2_BUCKET
CLOUDFLARE_R2_PUBLIC_URL
```

建议值：

```text
ADMIN_EMAIL=你的邮箱
ADMIN_PASSWORD=你的后台密码
RESEND_API_KEY=你的 Resend API Key
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_MODEL=deepseek-chat
NEXT_PUBLIC_WHATSAPP_NUMBER=你的 WhatsApp 号码
NEXT_PUBLIC_SITE_URL=你的正式网站地址
CLOUDFLARE_R2_ACCOUNT_ID=你的 Cloudflare Account ID
CLOUDFLARE_R2_ACCESS_KEY_ID=你的 R2 Access Key ID
CLOUDFLARE_R2_SECRET_ACCESS_KEY=你的 R2 Secret Access Key
CLOUDFLARE_R2_BUCKET=jump-rope-site-images
CLOUDFLARE_R2_PUBLIC_URL=你的 R2 图片公开地址
```

说明：

- `ADMIN_PASSWORD` 必须改成你自己的。
- `RESEND_API_KEY` 不填也能保存询盘，但不会邮件通知。
- `DEEPSEEK_API_KEY` 不填也能看询盘，但后台翻译按钮不能用。
- `NEXT_PUBLIC_SITE_URL` 上线后要改成正式域名。
- R2 这 5 个变量建议都填写，这样后台内容、询盘和图片都能持久保存。
- `CLOUDFLARE_R2_PUBLIC_URL` 必须能公开访问，否则后台上传图后前台打不开。

## 8. wrangler.jsonc 里需要确认的内容

项目根目录有：

```text
wrangler.jsonc
```

重点确认：

```jsonc
"name": "fitgearoem"
```

这个名称必须和 Cloudflare 后台项目名一致。你截图里的报错就是因为之前配置成了 `jump-rope-b2b-site`，但 Cloudflare 后台期望 `fitgearoem`。

当前配置已经移除了 KV 占位 ID，不会再出现：

```text
KV namespace 'REPLACE_WITH_SITE_DATA_KV_ID' is not valid
```

## 9. 第一次部署

Cloudflare 后台配置好后，点击：

```text
Save and Deploy
```

Cloudflare 会自动：

1. 从 GitHub 拉代码
2. 安装依赖
3. 执行构建命令
4. 发布网站

以后你只要本地 `git push`，Cloudflare 就会自动重新部署。

## 10. 部署后检查

上线后打开这些页面：

```text
/
/admin/login
/admin/content
/admin/inquiries
```

检查：

- 首页是否显示正常
- 后台是否能登录
- 后台是否能修改文字、图片、联系方式
- 上传图片是否能进入 R2
- 前台询盘表单是否能提交
- 后台是否能看到询盘
- 询盘状态是否能修改
- 翻译按钮是否可用
- 邮件通知是否正常

## 11. 常见问题

### 1. 首页可以打开，但后台/API 不能用

大概率是按静态 Pages 方式部署了。

这个项目需要完整 Next.js 后台能力，请确认 Cloudflare 构建方式是 OpenNext / Workers full-stack Next.js，而不是纯静态输出。

### 2. 后台登录失败

检查环境变量：

```text
ADMIN_PASSWORD
```

修改后重新部署。

### 3. 后台保存内容后刷新丢失

优先检查 R2 API 环境变量：

```text
CLOUDFLARE_R2_ACCOUNT_ID
CLOUDFLARE_R2_ACCESS_KEY_ID
CLOUDFLARE_R2_SECRET_ACCESS_KEY
CLOUDFLARE_R2_BUCKET
```

如果你选择用 KV，则检查 KV 绑定名称：

```text
SITE_DATA
```

### 4. 图片上传失败

检查 R2 绑定：

```text
SITE_IMAGES
```

如果你没有做 R2 绑定，就检查 R2 API 环境变量：

```text
CLOUDFLARE_R2_ACCOUNT_ID
CLOUDFLARE_R2_ACCESS_KEY_ID
CLOUDFLARE_R2_SECRET_ACCESS_KEY
CLOUDFLARE_R2_BUCKET
```

检查 R2 公开地址：

```text
CLOUDFLARE_R2_PUBLIC_URL
```

### 5. 图片上传成功但前台打不开

说明 R2 图片没有公开访问。

去 Cloudflare R2 后台给 `jump-rope-site-images` 开启公开访问或绑定自定义域名。

### 6. GitHub push 后 Cloudflare 没自动部署

检查：

- Cloudflare 是否已经绑定 GitHub
- 绑定的仓库是否是 `macdaox/fitgearoem`
- 绑定分支是否是 `main`
- Cloudflare 项目里是否开启自动部署
