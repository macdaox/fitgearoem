## 3. 登录 Cloudflare

先登录 Cloudflare：

```bash
npx wrangler login
```

命令会打开浏览器，登录你的 Cloudflare 账号并授权 Wrangler。

登录成功后可以检查：

```bash
npx wrangler whoami
```

## 4. 创建 Cloudflare KV

KV 用来保存：

- 后台编辑的网站文字
- 后台编辑的联系方式
- 询盘记录
- 询盘状态

创建 KV：

```bash
npx wrangler kv namespace create SITE_DATA
```

命令执行后会返回一段类似这样的内容：

```json
{
  "binding": "SITE_DATA",
  "id": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
}
```

把这里的 `id` 复制下来，打开 `wrangler.jsonc`，替换：

```jsonc
"id": "REPLACE_WITH_SITE_DATA_KV_ID"
```

改成：

```jsonc
"id": "你的KV_ID"
```

## 5. 创建 Cloudflare R2

R2 用来保存后台上传的图片。

创建网站图片 bucket：

```bash
npx wrangler r2 bucket create jump-rope-site-images
```

创建 OpenNext 缓存 bucket：

```bash
npx wrangler r2 bucket create jump-rope-b2b-site-opennext-cache
```

这两个名字已经写在 `wrangler.jsonc` 里，不建议随便改。

## 6. 配置 R2 图片公开访问

后台上传图片后，前台需要能访问图片，所以 R2 需要设置一个公开访问域名。

在 Cloudflare 后台操作：

1. 进入 Cloudflare 控制台
2. 进入 `R2 Object Storage`
3. 找到 `jump-rope-site-images`
4. 进入 bucket 设置
5. 找到公开访问或自定义域名设置
6. 开启公开访问，或者绑定一个自定义域名

得到公开地址后，打开 `wrangler.jsonc`，修改：

```jsonc
"CLOUDFLARE_R2_PUBLIC_URL": "https://YOUR_PUBLIC_R2_DOMAIN"
```

例如：

```jsonc
"CLOUDFLARE_R2_PUBLIC_URL": "https://images.yourdomain.com"
```

不要在结尾加 `/`。

## 7. 修改网站地址和 WhatsApp

打开 `wrangler.jsonc`，找到：

```jsonc
"vars": {
  "NEXT_PUBLIC_WHATSAPP_NUMBER": "15551234567",
  "NEXT_PUBLIC_SITE_URL": "https://jump-rope-b2b-site.YOUR_WORKERS_SUBDOMAIN.workers.dev",
  "DEEPSEEK_MODEL": "deepseek-chat",
  "CLOUDFLARE_R2_PUBLIC_URL": "https://YOUR_PUBLIC_R2_DOMAIN"
}
```

需要修改：

- `NEXT_PUBLIC_WHATSAPP_NUMBER`：你的 WhatsApp 号码
- `NEXT_PUBLIC_SITE_URL`：你的正式网站地址
- `CLOUDFLARE_R2_PUBLIC_URL`：你的 R2 图片公开地址

## 8. 设置 Cloudflare Secret

Secret 是敏感信息，不要写进 GitHub。

设置后台邮箱：

```bash
npx wrangler secret put ADMIN_EMAIL
```

设置后台密码：

```bash
npx wrangler secret put ADMIN_PASSWORD
```

设置 Resend 邮件 API Key：

```bash
npx wrangler secret put RESEND_API_KEY
```

设置 DeepSeek 翻译 API Key：

```bash
npx wrangler secret put DEEPSEEK_API_KEY
```

说明：

- `ADMIN_PASSWORD` 必须设置，不要用默认密码。
- `RESEND_API_KEY` 不设置也能收询盘，但不会邮件通知你。
- `DEEPSEEK_API_KEY` 不设置也能看询盘，但后台翻译按钮不能用。

## 11. GitHub 自动部署

项目已经有 GitHub Actions 文件：

```text
.github/workflows/deploy-cloudflare.yml
```

这个文件会在你 push 到 `main` 后自动部署到 Cloudflare。

你需要在 GitHub 仓库里添加两个 Secrets：

```text
CLOUDFLARE_ACCOUNT_ID
CLOUDFLARE_API_TOKEN
```

设置位置：

```text
GitHub 仓库 -> Settings -> Secrets and variables -> Actions -> New repository secret
```

`CLOUDFLARE_ACCOUNT_ID`：

- 在 Cloudflare 控制台右侧可以找到 Account ID。

`CLOUDFLARE_API_TOKEN`：

- 在 Cloudflare 创建 API Token。
- 需要有 Workers 部署权限。
- 需要能访问项目用到的 KV/R2 资源。

添加完成后，只要执行：

```bash
git push
```

GitHub 就会自动部署。

## 13. 常见问题

### 后台登录不了

检查 Cloudflare Secret 是否设置了：

```bash
npx wrangler secret put ADMIN_PASSWORD
```

重新设置后再部署一次：

```bash
npm run cf:deploy
```

### 后台保存内容后刷新丢失

检查 `wrangler.jsonc` 里的 `SITE_DATA` KV ID 是否正确。

### 图片上传成功但前台打不开

检查：

- R2 bucket 是否是 `jump-rope-site-images`
- R2 是否配置了公开访问域名
- `CLOUDFLARE_R2_PUBLIC_URL` 是否填写正确

### GitHub Actions 部署失败

检查 GitHub Secrets：

- `CLOUDFLARE_ACCOUNT_ID`
- `CLOUDFLARE_API_TOKEN`

也可以先在本地执行：

```bash
npm run cf:build
```

如果本地能构建，通常就是 GitHub 或 Cloudflare 权限配置问题。
